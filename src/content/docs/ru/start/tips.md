## Авто-перезагрузка во время правок

`/am reload` пересобирает все меню с диска, но набирать его после каждой правки быстро надоедает. Запусти `/am serve` - плагин начнёт следить за папкой меню, и каждое сохранение перезагружает изменённый файл. Чтобы выключить - запусти `/am serve` ещё раз.

:::caution
Только на время разработки. На проде не держи file watcher включённым - это лишний слушатель файловой системы, который не нужен, когда меню уже не меняются.
:::

## Горячая загрузка аддонов

Закинь новый jar с `addon.conf` в `plugins/AbstractMenus/addons/` и запусти:

```text
/am addons rescan
```

Плагин подхватит новый jar без перезапуска сервера. Чтобы обновить уже загруженный аддон после замены jar, используй `/am addons reload <name>`.

Аддоны как плагины требуют обычный `/reload` или перезапуск сервера - ими управляет Bukkit.

## MiniMessage, когда нужен

Если нужны `<click:run_command:...>`, `<hover:show_text:...>`, градиенты или радужный текст - поставь `useMiniMessage: true` в `config.conf`. По умолчанию выключено, чтобы не гонять парсер на каждый рендер. Включение даёт небольшую просадку производительности на покадровых обновлениях меню.

## CLI для переменных

Те же данные, что меню читают через `%var_*%` и `%varp_*%`:

```text
/var set kit_cooldown 1700000000000     # миллисекунды эпохи
/varp set Notch coins 500
/varp inc Notch coins 25
```

`inc`, `dec`, `mul`, `div` работают на числовых значениях. Они дешёвые. Запускай их из активатора `/say` (только из консоли) - так удобно применять побочные эффекты с ограничением частоты.
