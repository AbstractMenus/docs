First public alpha of the 2.0 line. The big change is a real extension API: built-in actions, rules, providers etc. now register through the same SPI external addons use. Operators get a drop-in upgrade with a few new commands; anyone who wrote against the 1.x `Types` / `Handlers` / `AbstractMenusProvider` static facades has work to do — see the [migration guide](/docs/developers/migration/).

## Highlights since 1.18

### Addon system

Two delivery paths:

- **Path 1 — plugin-as-addon.** A standard Bukkit plugin that depends on AbstractMenus and registers types through the API in its `onEnable`.
- **Path 2 — AM-loaded addon.** A small jar with an `addon.conf`, dropped into `plugins/AbstractMenus/addons/`. AbstractMenus loads it through an isolated classloader.

`/am addons` ships with `list`, `info`, `reload`, `load`, `rescan`, plus tab-complete. Path 1 plugins and the built-in core show up tagged accordingly.

Full guide: [Addon delivery](/docs/developers/addons/).

### Provider registry

`api.providers()` exposes five typed sections: `economy()`, `permissions()`, `levels()`, `placeholders()`, `skins()`. Each is a `ProviderSection<T>` with `register(id, handler, priority, owner)` and `resolve()` / `resolve(id)` lookups. Multiple providers per section coexist; operators pick the default in `config.conf`, menus override per-action with `provider: "..."`. The pre-2.0 static `Handlers` facade is gone.

Full guide: [Provider handlers](/docs/developers/handlers/).

### Folia

Entity-touching scheduled work now runs on the right region thread. 1.18 worked on Folia by accident in some places and crashed in others — this release fixes the crashes.

### Smaller jar

Plugin jar shrank from ~2.2 MB to ~1.1 MB. Adventure isn't bundled anymore — Paper already ships it.

### Other

- `/reload` no longer takes the plugin down.
- `MiniMessage` opt-in via `useMiniMessage: true` in `config.conf`.
- Click debounce floors against duplicate-click packets the Minecraft client emits per physical click.
- Various addon-lifecycle fixes around cross-addon registration interference, classloader leaks on hot-reload, transitive missing-dep chains, edge-case crashes when a player quits mid-close.

## Migrating from 1.x

Every old API call has a 2.0 replacement. The mapping table is at [Migration 1.x → 2.0](/docs/developers/migration/). For menu authors and server operators, HOCON configs are unchanged — your menus load as-is.

## Known limitation

A malicious addon can poke at private internals via reflection. Not patched here, would need JPMS-style isolation. Only relevant if you install untrusted third-party addons.
